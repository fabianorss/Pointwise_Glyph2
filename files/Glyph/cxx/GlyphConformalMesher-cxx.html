pw::ConformalMesher

The conformal mesher mode type

Derives From

pw::Object pw::Mode

Notes

This mode is used to create conformal domains on database models.

To create a mode of this type, use the pw::Application.begin command with the mode ConformalMesher specified, like this:

set mesher [pw::Application begin ConformalMesher $models]

MinimumEdgeLength

$mesher get/setMinimumEdgeLength ?-quilt quilt_list? tol

This attribute is the minimum edge length to use when creating triangles.

Parameters

-quilt quilt_listIf absent, the attribute is the global minimum edge length used when an explicit value is not assigned to a quilt.  If specified, quilt_list should be one or mor pw.Quilt objects.  When getting the minimum edge length, the return value will be a single value if a single quilt was specified (or if -quilt was not present), or a list of values corresponding to the list of quilts given.  When setting the minimum edge length, the value will be assigned to each quilt in the list, overriding the global value.

Type

This attribute is a float with the range (0.0, infinity).

Default

The default attribute value is 1.0e-8 times the model size.

getSymmetryBoundaries

$mesher getSymmetryBoundaries ?-strict? ?symmetry_list?

This action returns the manifold database boundaries that have been identified as being on a symmetry plane.  They are grouped into closed loops.

Parameters

-strictThis optional parameter is only meaningful when a symmetry argument is specified.  In such a case, only loops that whose boundaries match the specified symmetry qualifier will be returned.  If the strict argument is not specified, then loops that have any boundary that matches the specified symmetry qualifiers will be returned.
symmetry_listThis optional argument is a list of one or more string values from the following list: MinimumX, MaximumX, MinimumY, MaximumY, MinimumZ, MaximumZ.  If the strict argument is used, then only loops that match contain at least one boundary on each of the specified symmetry planes and no boundaries on any symmetry plane not specified will be returned.  If the strict argument is not specified, then any loop that has at least one boundary on any of the specified symmetry planes will be returned.  If not specified, all loops will be returned, regardless of the strict setting.  The special case of specifying an empty list will return all manifold boundary loops that did not map to a symmetry plane.

Returns

This action returns a list of loops, where each entry in the list is a list of quilt boundary objects that form a loop.

Example

Code

# Consider a loop that lies on both the MinimumX and MinimumY
# symmetry planes. $mesher is a ConformalMesher mode.
set loops [$mesher getSymmetryBoundaries -strict MinimumX
puts "Only MinimumX loops         : [llength $loops]"
set loops [$mesher getSymmetryBoundaries MinimumX
puts "Any MinimumX loops          : [llength $loops]"
set loops [$mesher getSymmetryBoundaries -strict { MinimumX MinimumY }
puts "Only MinimumX+MinimumY loops: [llength $loops]"
foreach loop $loops {
  puts "Loop:"
  foreach boundary $loop {
     set quilt [lindex $boundary 0]
     set index [lindex $boundary 1]
     puts [format "  -20s : %2d" [$quilt getName] $index]
  }
}

Output

Any MinimumX loops          : 1
Only MinimumX+MinimumY loops: 1
Loop:

  quilt-2             : 4

  quilt-3             : 6

createGridEntities

$mesher createGridEntities

This action creates grid entities on the database models specified when the mode was created.  The current settings of the mode will control how the connectors and domains will be created.  Note that domains created with this method are particularly fragile, and any changes to the connectors may result in domains with entirely different characteristics.

Parameters

This command takes no arguments.

Returns

This action returns a list of domains created.

Example

Code

# $mesher is a ConformalMesher mode
set ents [$mesher createGridEntities]
foreach ent $ents {
  puts "  $ent ([$ent getName])"
}
