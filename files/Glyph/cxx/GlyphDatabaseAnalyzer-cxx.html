analyze

$analyzer analyze

This action performs all prescribed database analysis checks.

Parameters

None

Returns

This action returns the integer total number of database issues found.

Information

By default, all analysis checks are disabled.  Use setEnabled to enable the checks are interested in.

This command supports progress updates.

Example

Code

set analyzer [pw::DatabaseAnalyzer create]
$analyzer setEnabled FreeEdges true
$analyzer addEntity [pw::Database getAll -type pw::Model]
puts [$analyzer analyze]

Output

10

getIssueType

$analyzer getIssueType ?-full? index

Parameters

Returns

This action returns a list of string names corresponding to the type (and subtypes if -full is specified) of the issue.

Example

Code

set analyzer [pw::DatabaseAnalyzer create]
$analyzer setEnabled FreeEdges true
$analyzer addEntity [pw::Database getAll -type pw::Model]
$analyzer analyze
puts [set issueCount [$analyzer getIssueCount]]
for {set i 1} {$i <= $issueCount} {incr i} {
    puts [$analyzer getIssueType -full $i]
}

getIssueStencilXYZs

$analyzer getIssueStencilXYZs ?-type type_var? index

This action gets the XYZ values for the stencil of the analyzer issue.

Parameters

-type typeVarThis optional parameter is a string variable name that will be assigned with the type of stencil used to represent the analyzer issue.
indexThis parameter is the integer issue.

Returns

This action returns either a list of xyz vector values or a list of list of xyz vector values depending on the stencil type.  The list will be empty for analysis types that do not support an XYZ stencil.  Use getIssueStencilType to determine if XYZs are applicable.  For Corner stencils, the result will be a list of 3 xyz vector values: the corner point, the tangent (pointing away from the corner) of the boundary immediately before the corner, and the tangent (pointing away from the corner) of the boundary immediately after the corner.  For EndCap types, the result will be a list of 4 xyz vector values and represents data for one end cap: an end cap direction, both boundary points, and the end cap apex.

See Also

getIssueStencilType, getIssueCount

getIssueStencilEntities

$analyzer getIssueStencilEntities index

This action gets the stencil entities associated with the analyzer issue.

Parameters

indexThis parameter is the integer issue.

Returns

This action returns a list of pw::DatabaseEntity or boundary objects depending on the issue type.  For PinchPoint types, the list will be two boundaries: the boundary immediately before the pinch point, and the boundary immediately after the pinch point.  Use getIssueStencilXYZs for stencil information.  For type FreeEdges, NonOrthogonalThinSurfaces, and DenseSurfaces, the return type will be a list of boundary objects.

getIssueType, getIssueStencilType, getIssueCount

getIssueStatistics

$analyzer getIssueStatistics ?-filter types? index

This action gets any statistics associated with the the analyzer issue.

Parameters

-filter typesThis optional parameter is a string list of statistic type names.  If provided, only statistics matching the filter type names will be returned.
indexThis parameter is the integer issue.

Returns

This action returns a list of lists of statistical data.  The first value in each statistical data is a string indicating the type of statistical data.  The type can be used to interpret the remaining data.

Information

PinchPointTurningAngleThe data is a single float.
PinchPointAssemblyToleranceThe data is a single float.
LoopPerimeterLengthThe data is a single float.
SlitMaxDistanceThe data is a single float.
SlitMaxSurfaceBendThe data is a single float.
SlitLengthThe data is a single float.
OffSymmetryDistanceThe data is a single float.  For every symmetry plane that other boundaries from the loop containing the issue boundaries belong to, the maximum distance to that symmetry plane is calculated for the boundaries in this issue.  The minimum of those distances is reported here.
MaxAssemblyToleranceThe data is a single float.
LeftEntityNameThe data is a string.
RightEntityNameThe data is a string.
OffSymmetryPlaneThe data is a string of the form [+-][XYZ] indicating which symmetry plane the OffSymmetryDistance refers to.
ThinSurfaceLengthThe data is a single float.
ThinSurfaceWidthThe data is a single float.
ThinSurfaceLengthRatioThe data is a single float.
ThinSurfaceSideThe data is a string indicating the end cap side.
SliversMaxDistanceThe data is a single float.
DenseSurfacePatchCountThe data is a single integer.
DenseSurfaceDeviationThe data is a single float.
DenseSurfacePatchReductionPercentThe data is a string.
RejectEntsThe data is a string.
RejectReasonThe data is a string.
RejectLocationsThe data is a list of vector.
RejectRelatedThe data is a list of string.

Example

Code

set analyzer [pw::DatabaseAnalyzer create]
$analyzer setEnabled FreeEdges true
$analyzer addEntity [pw::Database getAll -type pw::Model]
$analyzer analyze
set issueCount [$analyzer getIssueCount]
for {set i 1} {$i <= $issueCount} {incr i} {
    puts "Issue: $i"
    set statistics [$analyzer getIssueStatistics $i]
    if {[llength $statistics] == 0} {
        puts " No Statistic data"
    } else {
        foreach statistic [$analyzer getIssueStatistics $i] {
        puts " $statistic"
        }
    }
}

getSymmetryPlane

$analyzer getSymmetryPlane side

This action returns a pwu::Plane representing a symmetry plane.  This method is only valid after an analysis has been performed.

Parameters

sideThis required parameter specifies which symmetry plane is being queried.  Valid values are “+X”, “-X”, “+Y”, “-Y”, “+Z”, or “-Z”.

Returns

This action returns a pwu::Plane object representing the requested symmetry plane (which is the 4 coefficients of the plane equation of the form Ax + By + Cz = D).  The normal of the plane will point into the volume.

PinchPointMaxTurningAngle

$analyzer get/setAttribute PinchPointMaxTurningAngle < Default | angle >

This is the maximum turning angle allowed before a corner is considered a pinch point.

Type

This attribute is either the string “Default” or a float in the range [150, 180] indicating the max turning angle.

Default

The default value is 170.0.

FreeEdgesMode

$analyzer get/setAttribute FreeEdgesMode < Default | Normal | Basic >

This is the operation mode that is used by the FreeEdge analyzer.

Type

This attribute is the string “Default”, the string “Normal”, or the string “Basic”.

Information

When set to “Normal” mode, all free edge analysis will run.  When in “Basic” mode, some analysis will be skipped (e.g., slit detection will not be performed).

Default

The default value is “Normal”.
